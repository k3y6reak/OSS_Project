<p style="margin: 0 0 1.1em; line-height: 1.6;">이 장에서는 angr의 context sensitivity나  Function Manager와 같은 중요한 개념을 살펴보면서  angr의 <strong style="font-weight: bold; line-height: 1.6;">CFGAccurate</strong> 분석에 대해 자세하게 알아볼 것입니다.</p>

<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">개요</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">바이너리에서 할 수 있는 가장 기본적인 분석 방법은 <strong style="font-weight: bold; line-height: 1.6;">Control Flow Graph</strong>입니다. 이 CFG는 바이너리에서 jumps/calls/rets/etc 등 실행의 분기가 일어나는 basic blocks를 시각적으로 표현하는 그래프입니다. <br/>
angr에서는 <em style="line-height: 1.6;">fast CFG</em>와 <em style="line-height: 1.6;">accurate CFG(CFGAccurate)</em> 두 가지 형태의 CFG를 생성할 수 있습니다.  이름에서 유추할 수 있듯이, fast CFG를 생성하는 것은 일반적으로 accurate 방식보다 더 빠릅니다. 일반적으로 CFGFast가 사용자들이 원하는 형태일 것입니다. 이 페이지는 CFGAccurate에 대해 살펴볼 것입니다.</p>

<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">Context Sensitivity Level</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">angr은 모든 basic block을 실행하고 그 블록들을 검토하면서 CFG를 생성합니다. 이로 인해 몇몇 문제점들이 발생합니다.  basic block은 다른 context들에서 다르게 작동할 수 있습니다.  예를 들어, block이 함수의 return으로 종료된다면 해당 basic block을 포함하는 함수의 다른 caller에 따라 return하는 대상이 달라집니다.</p>

<p style="margin: 0 0 1.1em; line-height: 1.6;">context sensitivity level은 개념적으로 caller가 callstack을 유지할 수 있는 수입니다. 이 개념을 설명하기 위해서 아래 코드를 살펴보겠습니다.</p>

<p style="margin: 0 0 1.1em; line-height: 1.6;">context sensitivity level을 올리게 되면 CFG로부터 더 많은 정보를 얻을 수 있다는 장점이 있습니다. 예를 들어, context sensitivity가 1일 경우 CFG는 alpha에서 호출될 때 puts는 alpha를 return하며, error가 호출되면 puts에서 alpha를 return해줍니다. context sensitivity가 0일 경우 CFG는 간단하게 puts는 alpha, beta, 그리고 error를 return해줍니다. 이것은 구체적으로 IDA에서 사용되는 context sensitivity level입니다. context sensitivity level을 높이게 되면 분석 시간이 기하급수적으로 증가한다는 단점이 있습니다.</p>



<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">Using the CFG</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">CFG의 코어는 <a href="https://networkx.github.io/" style="background: transparent; color: #1980e6; text-decoration: none;" target="_blank">NetworkX</a> di-graph입니다. 즉, 모든 일반 NetworkX API들을 사용할 수 있다는 말이 되겠죠.</p>



<pre style="font-family: 'Source Code Pro',monospace; white-space: pre-wrap; display: block; background-color: rgba(102,128,153,0.05); color: #333; word-wrap: break-word; font-size: .9em; background: #f6f6f6; line-height: 1.6; margin: 0 0 1.1em; padding: 0; border: 0; border-radius: 5px; text-align: start; word-break: break-all;"><code style="font-family: 'Source Code Pro',monospace; font-size: inherit; background-color: transparent; white-space: pre-wrap; border-radius: 0; color: #f8f8f2; display: block; background: #23241f; padding: 1.3em 2em;"><div style="line-height: 1.6;"><span style="display: inline; line-height: 1.6;">&gt;&gt;&gt; </span><span style="display: inline; line-height: 1.6; color: #f92672;">print</span> <span style="display: inline; line-height: 1.6; color: #e6db74;">&quot;This is the graph:&quot;</span>, cfg.graph
</div><div style="line-height: 1.6;"><span style="display: inline; line-height: 1.6;">&gt;&gt;&gt; </span><span style="display: inline; line-height: 1.6; color: #f92672;">print</span> <span style="display: inline; line-height: 1.6; color: #e6db74;">&quot;It has %d nodes and %d edges&quot;</span> % (len(cfg.graph.nodes()), len(cfg.graph.edges()))

<p style="margin: 0 0 1.1em; line-height: 1.6;">CFG 그래프의 노드는 CFGNode 클래스의 인스턴스입니다. context sensitivity로 인해 주어진 basic block은 그래프에서 다중 노드를 가질 수 있습니다.(다중 context의 경우)</p>

<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">Viewing the CFG</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">CFG의 렌더링은 어려운 문제입니다. angr은 CFG 분석 결과를 렌더링하기 위한 built-in 메커니즘을 제공하지 않습니다. 그리고 matplotlib와 같은 기존의 그래프 렌더링 라이브러리를 사용하려고 시도한다면 이미지를 사용할 수 없는 결과를 초래하게 됩니다. <br/>
angr CFG를 보기 위한 하나의 솔루션은 <a href="https://github.com/axt/angr-utils" style="background: transparent; color: #1980e6; text-decoration: none;" target="_blank">axt’s angr-utils repository</a>입니다.</p>



<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">Shared Libraries</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">CFG 분석은 서로 다른 이진 객체의 코드 흐름을 구분하지 않습니다. 즉, 이것은 기본적으로 로드된 공유 라이브러리를 통해 control flow를 분석하려고 시도한다는 것입니다. 분석 시간을 며칠로 연장할 것이기 때문에 이것은 의도된 행동이 아닙니다. 공유 라이브러리 없이 바이너리를 로드하기 위해서는 Project constructor에 다음의 키워드 인자를 추가하세요: <code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">load\_options={'auto\_load\_libs': False}</code></p>



<h3 style="font-family: inherit; font-weight: bold; color: inherit; margin-top: 21px; margin-bottom: 10.5px; font-size: 27.2px; margin: 1.2em 0 .6em 0; text-align: start; line-height: 1.6;">Function Manager</h3>

<p style="margin: 0 0 1.1em; line-height: 1.6;">CFG의 결과는 <code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">cfg.kb.functions</code>를 통해 접근할 수 있는 Function Manager라고 불리는 오브젝트를 생성합니다. 이 객체를 가장 일반적으로 사용하는 경우는 dictionary와 같은 것에 접근하는 것입니다. 이것은 주소를 <code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">Function</code> 객체에 매핑합니다. 이 객체는 함수에 대한 속성을 알려줍니다.</p>



<pre style="font-family: 'Source Code Pro',monospace; white-space: pre-wrap; display: block; background-color: rgba(102,128,153,0.05); color: #333; word-wrap: break-word; font-size: .9em; background: #f6f6f6; line-height: 1.6; margin: 0 0 1.1em; padding: 0; border: 0; border-radius: 5px; text-align: start; word-break: break-all;"><code style="font-family: 'Source Code Pro',monospace; font-size: inherit; background-color: transparent; white-space: pre-wrap; border-radius: 0; color: #f8f8f2; display: block; background: #23241f; padding: 1.3em 2em;"><div style="line-height: 1.6;"><span style="display: inline; line-height: 1.6;">&gt;&gt;&gt; </span>entry_func = cfg.kb.functions[b.entry]
</div></code></pre>

<p style="margin: 0 0 1.1em; line-height: 1.6;">Functions에는 몇가지 중요한 속성이 있습니다.</p>

<ul style="margin-top: 0; margin-bottom: 1.1em; line-height: 1.6;"><li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.block_addrs</code>는 함수에 속하는 basic block이 시작하는 주소들의 집합입니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.blocks</code>는 capstone을 사용하여 디스어셈블하고 탐색할 수 있는 함수에 속한 basic block의 집합입니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.string_references()</code>는 함수의 어느 지점에서든 참조된 모든 상수 문자열들의 리스트를 반환합니다. 그것들은 (addr, string) 튜플의 형태인데, addr은 바이너리의 데이터 섹션에 있는 주소이고, strings는 문자열의 값을 포함하는 python string입니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func_returning</code>은 함수가 return할 수 있는지 없는지 검증하는 boolean 값입니다. False는 모든 경로가 반환되지 않는 것을 나타냅니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.callable</code>은 이 함수를 참조하는 angr Callable 객체입니다. python 인자를 가진 python 함수와 같이 호출할 수 있으며, 이러한 인수를 사용하여 함수를 실행한 것처럼 실제 결과(symbolic일 수 있다)를 얻어낼 수 있습니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.transition_graph</code>는 함수 자체 내의 control flow를 설명하는 NetworkX DiGraph입니다. IDA가 기능별 수준에서 표시해주는 CFG와 비슷합니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.name</code>은 함수의 이름입니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.has_unresolved_calls</code>와 <code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry.has_unresolved_jumps</code>는 CFG 내의 부정확성을 감지하는 것과 관련이 있습니다. 때대로 간접 호출이나 jump가 가능한 대상이 무엇인지 감지할 수 없습니다. 만약 함수 내에서 발생한다면 그 함수는 적절한 <code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">has_unresolved_*</code>의 값을 True로 세팅합니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.get_call_sites()</code>는 다른 함수의 호출에서 끝의 basic block에 해당하는 모든 주소들의 집합을 반환합니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.get_call_target(callsite_addr)</code>은 callsite_addr이 call site address의 리스트에서 주어질 경우 해당 callsite가 호출할 위치를 반환합니다.</li>
<li style="line-height: 1.6;"><code style="font-family: 'Source Code Pro',monospace; font-size: .9em; padding: 2px 4px; color: #c7254e; background-color: #f9f2f4; white-space: normal; border-radius: 4px;">entry_func.get_call_return(callsite_addr)</code>은 callsite_addr이 call site address의 리스트에서 주어질 경우 해당 callsite가 반환될 위치를 반환합니다.</li>
</ul>
